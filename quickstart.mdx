## Prerequisites

Before you begin, ensure you have:

- **Node.js** 18.17.0 or higher
- **npm** 9.0.0 or higher
- **PostgreSQL** (local or hosted)
- **AWS account** with S3 (uploads) and SES (email) configured
- **Redis** (optional; for BullMQ campaign scheduling)

## Step 1: Clone and install

<Steps>
  <Step title="Clone the app">

The app lives in the `mailapp` directory. If you have it elsewhere, clone or open that repo:

```bash
cd /path/to/mailapp
```

  </Step>
  <Step title="Install dependencies">

```bash
npm install
```

  </Step>
</Steps>

## Step 2: Environment setup

Create a `.env` file in the project root (use `.env.example` if provided):

```env
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/vibedropper"

# NextAuth
NEXTAUTH_SECRET="generate-a-random-secret-here"
NEXTAUTH_URL="http://localhost:3000"

# AWS S3
AWS_ACCESS_KEY_ID="your-aws-access-key"
AWS_SECRET_ACCESS_KEY="your-aws-secret-key"
AWS_REGION="us-east-1"
AWS_S3_BUCKET="your-bucket-name"

# AWS SES
AWS_SES_REGION="us-east-1"
AWS_SES_FROM_EMAIL="noreply@yourdomain.com"
AWS_SES_FROM_NAME="Vibedropper"

# Google OAuth (optional)
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"

# App
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NEXT_PUBLIC_APP_NAME="Vibedropper"

# Redis (optional)
REDIS_URL="redis://localhost:6379"

# Topol.io email editor — required for campaigns
NEXT_PUBLIC_TOPOL_API_KEY="your-topol-api-key"
```

Generate a strong secret for `NEXTAUTH_SECRET`:

```bash
openssl rand -base64 32
```

<Tip>
**Topol.io**: Get an API key from [app.topol.io](https://app.topol.io) → **Settings** → **Plugin** → **New API Token**. Whitelist `localhost:3000` and your production domain.
</Tip>

## Step 3: Database setup

<Steps>
  <Step title="Generate Prisma client and push schema">

```bash
npx prisma generate
npx prisma db push
```

  </Step>
  <Step title="Seed the database (optional)">

```bash
npm run db:seed
```

After seeding, you can log in with:

- **Email**: `demo@vibedropper.com`
- **Password**: `demo123456`

  </Step>
</Steps>

## Step 4: Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Sign up or use the demo credentials after seeding.

<Note>
The app uses Next.js with Turbopack for fast local development. Changes hot-reload automatically.
</Note>
